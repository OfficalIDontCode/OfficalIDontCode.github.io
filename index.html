## about

i started on C and assembly. first real project was a windows kernel driver, badly written, that bluescreened my pc maybe forty times before it loaded once. i was hooked. spent the next few years reading other people's code and trying to figure out how anti-cheats worked.

reverse engineering taught me to look at software the way attackers do. that's still the lens i use. when i write game code i'm thinking about exploits, race conditions, what happens if a player's framerate drops to zero mid-action. it's not paranoia, it's just where my brain goes by default now.

i moved to roblox because i wanted to build whole games on my own. luau is fast, the engine is solid, the userbase is huge, and tooling like rojo and roblox-ts let me work the way i want to work. i've been on it about a year and a half. paid commissions most of that time.

i answer fast and i don't drop projects. if you want to know what i can do, the projects below are real things real people use.

~/portfolio/projects/primal-islands.md roblox · 2026

### primal islands

role: solo developer, systems lead

// click to play. ~25mb so it won't autoplay.

a survival sandbox. you spawn on procedural islands and have to gather, craft, and not die. built it in roblox-ts with flamework and a react-driven HUD. inventory and crafting are fully custom (no marketplace kit). the gathering side has weighted loot, tool-tier gating, and per-resource respawn timers.

the hardest part wasn't the gameplay loop. it was making saves survive players leaving in the middle of an action without losing items or duping them. ended up writing a session-locked profile layer on top of profileservice that holds the lock across yields and reconciles on rejoin. it's the single piece of code i'd port to every other game i make.

→ play (beta)

stack: roblox-ts · flamework · @rbxts/react · profileservice · rojo

~/portfolio/projects/grand-theft-city.md roblox · 2026

### grand theft city

role: solo developer

open-world city RP. inventory and hotbar that survive death and rejoin (items don't wipe, items don't dupe), a banking system called RothBank with deposits and withdrawals, full datastore-backed economy. main menu, inventory grid, ATM interface are all hand-built UI.

the fun problem here was making the inventory feel responsive when the datastore was being slow. i ended up doing optimistic local mutations that get reconciled when the write returns. drag and drop never feels laggy because the UI doesn't wait on the network for the obvious cases.

stack: roblox · custom inventory framework · rothbank ATM · datastore economy · custom UI

~/portfolio/projects/codedefender.md software protection · active

### codedefender

role: co-founder, back engineering labs

a software protection platform i co-founded under back engineering labs. binary rewriting, novel obfuscation, anti-tamper, program hardening for windows binaries. it actually supports modern windows security features (CET, CFG, ACG) instead of breaking them, which is the bar most other protectors fail at.

as far as i know it's the only protector that can rewrite ntoskrnl and chromium without crashing them. that sounds like a flex but it's mostly the result of years of fixing weird edge cases in PE parsing and instrumentation. C++ and x64.

→ codedefender.io

stack: c++ · x64 binary rewriting · PE tooling · anti-tamper · llvm / B.L.A.R.E.

~/portfolio/projects/oink.industries.md gmod tooling · active

### oink.industries

role: owner, engineer

a lua injector and tooling platform for garry's mod. auth, licensing, anti-tamper, a custom HTTP API. it's processed millions of sessions across live communities. GLua on the client, MySQL and redis on the backend, stripe for payments. i built it and i run it.

the part i'm most proud of is the integrity layer. everyone in the gmod cheating scene tries to break it and so far the design has held up. that's mostly because i was on the other side of that fight for years before i ever shipped a paid product.

→ oink.industries

stack: glua · lua injection · mysql · redis · stripe · http api
